# GhostTyper v1.3.0 (geplant) - Feature-Übersicht

Stand: 2026-02-20

- Aktuelle Paketversion im Repo: `1.2.0` (`package.json`)
- Die unten gelisteten Features sind als **nächster Release-Block (v1.3.0)** dokumentiert.
- Referenz im Changelog: `CHANGELOG.md` unter `Unreleased - Ziel: v1.3.0`.

## Enthaltene Features (v1.3.0)

### 1) Produktivität: One-Klick-Actions / Workflows

- Text-Assistent mit 1-Klick-Workflows:
  - Meeting -> Aktionsplan
  - Notizen -> Status-Update
  - Rohnotiz -> Kurzbericht
- API:
  - `GET /api/workflows`
  - `POST /api/workflows/execute`

### 2) Produktivität: Auto-Glossar

- Kontextbegriffe aus Nutzerhistorie vorschlagen und per Klick übernehmen.
- API:
  - `GET /api/glossary/suggestions`

### 3) Produktivität: Modellauswahl + Kostenvorhersage

- Empfehlung je Tasktyp (Text-AI, Übersetzung, Upload) mit Kosten-/Token-Prognose.
- API:
  - `POST /api/model-assistant`
  - Ampel-Status (`trafficLight`) für Startentscheidung (grün/gelb/rot)
  - Budgetinformationen inkl. Guardrail-Prognose

### 4) Team Realtime (Live)

- Neue Seite: `/realtime`
- Live-Funktionen:
  - Live-Transkript
  - Live-Dokument (Markdown)
  - Live-Wissensgraph
  - Team-Mitglieder mit Rollen (`owner`, `editor`, `viewer`)
  - Abschluss-Finalisierung mit `finalization_state` (idle/running/done/failed)
  - Duplikat-Erkennung für wiederholte Live-Chunks
- APIs:
  - `GET|POST /api/realtime/sessions`
  - `GET|PATCH /api/realtime/sessions/[id]`
  - `POST|DELETE /api/realtime/sessions/[id]/members`
  - `POST /api/realtime/sessions/[id]/ingest`
  - `GET /api/realtime/sessions/[id]/stream`

### 5) Tabellen-Vorlagen Editor V2 (UX-Redesign)

- Visueller, weniger technischer Builder:
  - Schnellstart-Presets
  - Spalten schnell anlegen
  - optionaler Expertenmodus für interne Keys
  - vereinfachte Formelhilfe und Live-Vorschau
- Ziel: Tabellen sauber definieren, damit das KI-Modell JSON zuverlässig befüllt.

### 6) Workflow-Editor mit Versionen/Rollback

- Eigene 1-Klick-Workflows direkt in den Einstellungen anlegen/bearbeiten.
- Versionen mit Notiz speichern und bei Bedarf rollbacken.
- APIs:
  - `POST /api/workflows`
  - `DELETE /api/workflows/[workflowId]`
  - `GET /api/workflows/[workflowId]/versions`
  - `POST /api/workflows/[workflowId]/rollback`

### 7) Budget-Guardrails pro Mitglied/Account

- Zusätzliches Setting: `member_monthly_budget_limit`.
- Vor KI-Aufrufen wird die prognostizierte Mehrkostenlast geprüft.
- Blockiert Aktionen, wenn die Prognose das Budget überschreitet.

### 8) Security/Ops: Audit-Log + Upload-Scan-Hook

- Kritische Aktionen werden im Audit-Log gespeichert.
- API:
  - `GET /api/audit-log`
- Upload-Pipeline mit optionalem Virus-Scan-Hook via Env-Konfiguration (`UPLOAD_VIRUS_SCAN_MODE`, `UPLOAD_VIRUS_SCAN_CMD`, ...).

## Stabilitäts-/Fixblock in diesem Release

- Tabellen-Analyse-Pipeline: `analysis_type` / `table_schema` Persistenz und API-Ausgabe korrigiert.
