# GhostTyper v1.2.0 - Neue Features

Stand: 2026-02-19

## Übersicht

Version 1.2.0 bringt zwei neue produktive Features zur Verbesserung der Organisation und Auffindbarkeit von Inhalten.

---

## 1. Volltext-Suche

### Beschreibung
Die Suche in der Historie wurde von einer reinen Dateinamen-Suche auf eine Volltext-Suche erweitert. Jetzt werden auch Transkript-Inhalte und KI-Analysen durchsucht.

### Verwendung
1. Historie-Seite öffnen (`/transcriptions`)
2. Suchbegriff im Suchfeld eingeben
3. Ergebnisse erscheinen nach 300ms Debounce mit Loading-Indicator

### Technische Details
- **Server-Side**: Suche läuft auf der Datenbank mit PostgreSQL `ILIKE`
- **Durchsuchte Felder**:
  - `original_name` (Dateiname)
  - `text` (Transkript-Inhalt)
  - `analysis::text` (KI-Analyse als JSON-String)
- **Debouncing**: 300ms Verzögerung für bessere Performance
- **Case-Insensitive**: `ILIKE` ignoriert Groß-/Kleinschreibung

### API
```
GET /api/transcriptions?search=meeting
```

### Frontend-Änderungen
- `pages/transcriptions.js`: Debounced Search mit `searching` State
- `lib/api.js`: `getTranscriptions(search)` mit optionalem Parameter

## 2. Vorlagen-Kategorien

### Beschreibung
Vorlagen können jetzt in selbst erstellten Kategorien organisiert werden. Dies ermöglicht eine bessere Strukturierung bei vielen Vorlagen.

### Verwendung
1. Einstellungen öffnen → "Verarbeitungstemplates"
2. Im oberen Bereich "Kategorien" neue Kategorie erstellen
3. Beim Bearbeiten einer Vorlage kann eine Kategorie zugewiesen werden

### Features
- **Kategorien erstellen**: Name eingeben, Enter drücken
- **Kategorien bearbeiten**: Stift-Icon klicken
- **Kategorien löschen**: X-Icon klicken (Vorlagen werden nicht gelöscht)
- **Farbcodierung**: Jede Kategorie hat eine Farbe (Standard: Orange)

### Technische Details
- **Datenbank**:
  ```sql
  CREATE TABLE template_categories (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,
    color VARCHAR(20) DEFAULT '#f97316',
    position INTEGER DEFAULT 0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
  );
  
  ALTER TABLE templates ADD COLUMN category_id INTEGER 
    REFERENCES template_categories(id) ON DELETE SET NULL;
  ```
- **API-Endpunkte**:
  - `GET /api/template-categories` - Alle Kategorien
  - `POST /api/template-categories` - Neue Kategorie
  - `PUT /api/template-categories/[id]` - Kategorie bearbeiten
  - `DELETE /api/template-categories/[id]` - Kategorie löschen

### Frontend-Änderungen
- `pages/settings.js`: Kategorien-Section mit CRUD-UI
- `lib/api.js`: Neue API-Funktionen für Kategorien

## Migration

Für bestehende Installationen muss die Datenbank einmalig aktualisiert werden:

```bash
curl -X POST http://localhost:3000/api/db-init \
  -H "x-init-secret: dev-db-init-secret"
```

Dies fügt die neue Tabelle und Spalte hinzu, ohne bestehende Daten zu beeinflussen.
